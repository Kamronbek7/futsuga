
## ✅ README.md Strukturasi — bo‘limlar tartibi

### 1. **Proyekt nomi va qisqacha tavsifi**

* Loyihaning rasmiy nomi (masalan: `Futsuga`)
* 1–2 gapda nima uchun kerakligi (slogan/sarlavha)

---

### 2. **Veb havolalar (Quick Links)**

* 🌐 Web-sayt (agar mavjud bo‘lsa)
* 📘 Qo‘llanma sahifasi (guide)
* 📦 Paket yoki o‘rnatish sahifasi (PyPI, npm, etc)

### 3. **O‘rnatish (Installation)**

* `pip install ...` yoki boshqa usullar
* GitHub’dan klonlash orqali o‘rnatish
* `.fsg` faylni ishga tushirish misoli

---

### 4. **Asosiy sintaksis misoli**

* `.fsg` fayldan qisqa va sodda misol
* Foydalanuvchi qanday ishlatadi — oddiy namunaviy kod

---

### 5. **Afzalliklar (Why Futsuga?)**

* No-code/low-code bo‘lishi
* Oddiy sintaksis
* Telegram API integratsiyasi
* Himoyalangan fayllar, ruxsat tizimi
* Har qanday foydalanuvchi uchun

---

### 6. **Skrinshotlar**

* Web editor, Telegram bot preview, CLI misollar
* (Agar hozircha mavjud bo‘lmasa, joy ajratib qo‘y)

---

### 7. **Qo‘llanma (Documentation / Guide)**

* Havola bilan: `docs/`, `futsuga.uz/docs`, `guide.pdf` va hokazo
* Boshlovchilar uchun alohida yo‘nalish: "Dasturlashdan xabari yo‘qlar"
* Tajribali dasturchilar uchun: `.fsg` → `.json` → `.py`, API qo‘llanma

---

### 8. **Hissa qo‘shish (Contributing)**

* Qanday yordam berish mumkin
* `issues` ochish, `pull request` yuborish
* Yangi qoida/sintaksis qo‘shish jarayoni

---

### 9. **Mualliflik va himoya mexanizmlari**

* `.fsg` faylning imzolanishi, shifrlanishi
* Koddan foydalanish mumkin, lekin tahrirlab bo‘lmaydigan holatlar
* Tokenni tashqi faylda saqlash va xavfsizlik masalalari

### 10. **Litsenziya**
